Historia

  • Home

Contenido:

En el año 2009 una desconocida empresa llamada Nitobi crea un framework multiplataforma para desarrollo móvil llamado Phonegap. La idea era crear aplicaciones HTML5 con la capa de negocios en Javascript, con el que pudieramos acceder a las funciones nativas y que fuera visto en cualquier sistema móvil. La idea triunfó y en 2011 donaron el código fuente a Apache, convirtiéndolo en Open Source. En octubre Adobe compra Nitobi por la transición de Flash a HTML5, a pesar de que el código se donará a Apache, en febrero del 2012 Apache renombra el producto como Cordova ya que Phonegap era registro de Adobe, lo renombraron así porque era el nombre de una calle en la que estaban las oficinas de Nitobi y finalmente en Septiembre del 2012 Adobe lanza Phonegap build para IOS, Android, Windows Phone, etc.

¿Qué es?

  • Home

Contenido:

Phonegap es un framework para el desarrollo de aplicaciones móviles producido por Nitobi, y comprado posteriormente por Adobe Systems.

Principalmente, PhoneGap permite a los programadores desarrollar aplicaciones para dispositivos móviles utilizando herramientas genéricas tales como Javascript, HTML5 y CSS3.

Características

  • Home

Las aplicaciones resultantes son híbridas, es decir que no son realmente aplicaciones nativas al dispositivo.

En la tercera versión de PhoneGap se incorpora el uso de una interfaz de comandos a través de consola.

Maneja API que permiten tener acceso a elementos como el acelerómetro, la cámara, los contactos en el dispositivo, la red, el almacenamiento, las notificaciones, etc.

Es una distribución de Apache Cordova.
